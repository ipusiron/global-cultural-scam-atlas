# Content Guidelines (GCSA)

## 基本方針
- 本プロジェクトは教育目的。特定の国・文化・人々を一般化しない。
- 表現は「攻撃者がその傾向を **悪用しうる**」という枠組みで統一。
- Red Flags ↔ Mitigations を必ず対で記述し、行動可能な短文で。

## 編集チェックリスト
- [ ] 断定・偏見を避けた表現
- [ ] 最低 `ja` を充足（可能なら `en` も）
- [ ] `attack_vector/targets/mediums/tags` は制御語彙内
- [ ] `risk_score` の根拠は `references` に寄せる
- [ ] URL 生存確認済み
- [ ] ID 重複なし（国コード+連番）
